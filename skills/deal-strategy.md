# Deal Strategy Workflow

Execute these steps IN ORDER. Complete each step fully and produce the required OUTPUT before moving to the next. Do not skip steps.

**Trigger phrases:** "deal strategy for [Company]" / "MEDDPICC [Company]" / "strategy for [Company]" / "where do we stand with [Company]"

---

## Step 1: Identify the Deal

Ask (if not already clear):
> Which deal are we running strategy on? And is there a specific trigger for this review? (Upcoming call, deal stalling, close date approaching, competition, etc.)

**OUTPUT required before proceeding:**

```
🎯 Deal Strategy Session — [Company]
Trigger: [Why we're doing this now]
ARR: $[X]k | Stage: [Stage] | Close Date: [Date]
```

---

## Step 2: Load Full Deal Context

Read:
- `second-brain/profile.md`
- `second-brain/company.md`
- `second-brain/deals.md` (this company's entry)
- `accounts/[company-name].md` (full history)

**OUTPUT required before proceeding:**

```
📁 Full context loaded for [Company]

Deal age: [How long in pipeline]
Last meaningful interaction: [Date + what happened]
Total interactions: [Count]
Key contacts: [Names + titles]
Stage history: [Progression through stages]
Previous MEDDPICC snapshot: [Prior assessment summary]
```

---

## Step 3: Full MEDDPICC Assessment

Conduct a thorough MEDDPICC analysis based on all available context.

**OUTPUT required before proceeding:**

```
🔍 MEDDPICC Analysis — [Company]

---

### M — Metrics
**Status:** 🟢 Strong / 🟡 Developing / 🔴 Weak / ⬜ Unknown

**What we know:**
- [Specific metrics or ROI that have been quantified]
- [Business outcomes the prospect has tied to this decision]

**What's missing:**
- [Gaps — e.g., "We have time savings but haven't tied to a dollar figure"]

**Recommended action:**
- [e.g., "Build a customized ROI model using their headcount and average salary data before next call"]

---

### E — Economic Buyer
**Status:** 🟢 / 🟡 / 🔴 / ⬜

**Who we believe it is:** [Name, Title — or "Unknown"]
**Have we met them:** [Yes / No / Indirect]
**Their priorities:** [What they care about based on role and what we know]
**Our access:** [Direct / Through champion / Blocked]

**What's missing:**
- [e.g., "We've never had a direct conversation with the EB — champion has been buffering us"]

**Recommended action:**
- [e.g., "Ask champion directly: 'What's the best way to get [EB name] visibility into what we're building together?'"]

---

### D — Decision Criteria
**Status:** 🟢 / 🟡 / 🔴 / ⬜

**What we know they're evaluating on:**
- [Criterion 1 — e.g., search quality across connectors]
- [Criterion 2 — e.g., security / permissions model]
- [Criterion 3 — e.g., time to value / deployment complexity]

**Informal vs. formal criteria:**
- [e.g., "IT team cares about security; end users care about UX; EB cares about ROI"]

**Where Glean wins on criteria:** [Our strengths vs. their criteria]
**Where we're at risk:** [Criteria where we may lose]

**Recommended action:**
- [e.g., "Validate that security is truly a gate — or a preference — before over-indexing on it"]

---

### D — Decision Process
**Status:** 🟢 / 🟡 / 🔴 / ⬜

**Steps we know:**
1. [e.g., Technical validation / POV — in progress]
2. [e.g., IT security review]
3. [e.g., Procurement / legal]
4. [e.g., Executive sign-off]

**Who's involved at each step:**
- [Step: Name / role]

**Timeline:** [Expected timeline based on what we know]
**Hidden steps we may not know about:**
- [e.g., "Have we asked if there's a budget approval process or competing priorities for IT?"]

**Recommended action:**
- [e.g., "Map the full process explicitly: 'If everything goes well, walk me through what happens between now and a signed contract.'"]

---

### I — Identify Pain
**Status:** 🟢 / 🟡 / 🔴 / ⬜

**Business pain (company level):**
- [e.g., "Knowledge silos slowing down product team — engineers spending 4hrs/wk searching"]

**Personal pain (individual level):**
- [e.g., "Champion is embarrassed by the current state — has escalated internally before"]

**Compelling event:**
- [e.g., "New CTO joined 3 months ago and has made AI tools a priority for H1"]
- [Or: "No compelling event identified yet — this is a risk"]

**Urgency level:** [High / Medium / Low — and why]

**Recommended action:**
- [e.g., "If no compelling event, create one: connect Glean to a board-level initiative they've mentioned"]

---

### C — Champion
**Status:** 🟢 Strong / 🟡 Developing / 🔴 Weak / ⬜ Unknown

**Who our champion is:** [Name, Title]
**Champion strength assessment:**
- Do they have power and influence? [Yes / No / Unclear]
- Do they sell internally for us when we're not in the room? [Evidence: ]
- Have they shared internal information (org chart, budget, competing priorities)? [Yes / No]
- Have they coached us on how to win? [Yes / No]
- Are they at risk if we lose? [Do they have skin in the game?]

**Champion health score:** [🟢 Strong advocate / 🟡 Friendly but passive / 🔴 May not survive internally]

**Recommended action:**
- [e.g., "Equip champion with an internal deck they can use to present to the EB"]
- [e.g., "Test champion: ask them to set up an introduction to [EB name]"]

---

### C — Competition
**Status:** 🟢 / 🟡 / 🔴 / ⬜

**Competitors in play:**
- [Competitor 1] — [How we know / what we know about their position]
- [Competitor 2] — [Same]

**Where Glean wins vs. each:**
- vs. [Competitor 1]: [Key differentiators]
- vs. [Competitor 2]: [Key differentiators]

**Where they could beat us:**
- [Honest assessment — e.g., "Microsoft will always win on price if bundled"]

**Recommended action:**
- [e.g., "Get champion to confirm we're the preferred vendor before investing more — or find out what's giving [Competitor] an edge"]
```

---

## Step 4: Deal Health Assessment

**OUTPUT required before proceeding:**

```
⚠️ Deal Health Assessment — [Company]

**Overall deal health:** 🟢 Healthy / 🟡 At risk / 🔴 In trouble

**Biggest strengths:**
1. [Strength 1]
2. [Strength 2]

**Critical gaps (must address to close):**
1. [Gap 1 — e.g., "No EB access — fatal if we don't fix before [date]"]
2. [Gap 2]

**Risks that could kill this deal:**
- [Risk 1 — e.g., "Microsoft bundling conversation happening at the board level"]
- [Risk 2]

**Is this deal closeable by [close date]?** [Yes / Likely / Unlikely — with reasoning]

**Forecast category:** Commit / Best Case / Pipeline
**Recommended adjustment:** [Should this stay in forecast, push out, or be recategorized?]
```

---

## Step 5: Strategic Recommendations

**OUTPUT required before proceeding:**

```
🧠 Strategic Recommendations — [Company]

**The deal in one sentence:**
[Honest framing of where things stand]

**The #1 thing to do in the next 7 days:**
[Single most important action — specific]

**The 3-move play:**
1. [Move 1 — immediate]
2. [Move 2 — this week]
3. [Move 3 — before close date]

**Who else should be involved (internal)?**
- [e.g., "Bring in your manager for an EB call — adds gravitas"]
- [e.g., "SE needs to prep a customized security deep-dive"]

**Questions Lauren should be asking but isn't:**
- [Uncomfortable but important question 1]
- [Uncomfortable but important question 2]

**What would make me more confident in this deal:**
- [Signal 1 — e.g., "EB agrees to a 30-min call"]
- [Signal 2 — e.g., "Champion sends internal summary without being asked"]
```

## Step 6: Update Account File and Deals.md

Update `accounts/[company-name].md` with today's strategy session notes.
Update `second-brain/deals.md` with revised MEDDPICC scores and recommended next action.

**OUTPUT required:**

```
✅ Strategy session complete for [Company].

Account file updated: accounts/[company-name].md
Deals.md updated: MEDDPICC status + next action refreshed

Want me to draft the [champion email / internal EB brief / ROI model / executive deck] to execute on the #1 recommendation?
```

**STOP and wait for direction.**
